Introduction to Recommender Systems - 1

Posted on 2021-10-27 Edited on 2021-11-02 In 추천시스템 , coursera

Introducing Recommender Systems

Preferences and Ratings

Preference model

  • 추천을 하기 위해서는 유저가 어떤 것을 좋아하는지, 연관성이 있는 데이터는 어떤 것들인지 등에 대한 데이터가 필요함.
    • What users tell us (ratings)
    • What users do (actions)

Explicit ratings

  • 직접적 선호도 : 사용자에게 직접 물어봐서 얻을 수 있음
    • 별점 : 별 반개, 별의 의미(5점 : 최고 등)
    • 좋아요 : Up/Down(Like, +1등 좋아요만 집계하는 경우도 있음). 수명이 짧은 아이템(게시글)등에 사용
    • 연속적인 스케일의 점수
    • 두 아이템간의 비교
    • Hybrid
  • Difficulties
    • Joke ratings : 노이즈가 존재
    • 시간에 사용자의 선호도가 바뀔 수 있음
    • 평점, 좋아요가 상황에 따라 어떤 의미인지가 다를 수 있음

Implicit data

  • 간접적 데이터 : 사용자의 행동에 의해 수집된 정보
    • Click : 평점은 주지 않아도 특정 종류의 기사를 많이 클릭했을 수 있음
    • Purchase : 구매도 마찬가지
    • Follow
  • Difficulties
    • 물건을 구매했지만, 마음에 들지는 않을 수 있음.
    • 클릭했다고 해서, 해당 게시글을 좋아하는건 아님
    • 또 위의 행동들(binary)를 어떻게 수치화 할지?

Predictions and Recommendations

비교

  • 추천은 사용자가 좋아할만한 아이템(혹은 아이템의 리스트)를 제공해 줌
Prediction Recommendation
pro 해당 아이템이 얼마나 좋을지 수치로 보여줌 여러 아이템들 중 좋아할 만한 선택지를 제시해 줌
con 잘못된 정보를 줄 수 있음 추천 아이템이 별로일 경우, 다른 선택지들의 정보를 보여주지 않음

추천은 얼마나 explicit 해야 하는가?

  • 너무 구체적으로 추천해주면 사람들이 잘 믿지 않음.
  • 구체적인 예측과 soft한 추천 사이의 균형을 잘 맞춰야 함.
  • Soft하게 여러 선택지를 제시해주고 그 중에서 사용자가 선택하도록 하면 좋음.

Taxonomy of Recommenders

  • 다음 8가지를 고려하여 Recommender를 분류
    • Domain, Purpose, Recommendation context, Whose opinions, Personalization level, Privacy and Trustworthiness, Interfaces, Recommendation Algorithms

Domain

  • 어떤 것을 추천하는지?
  • 기사(text)? 물건(product)? 음악(Sequence)? 등
  • 새로운 아이템(새로나온 영화, 책 등)을 추천하는지, 오래된 아이템(음악, 물건 등)을 다시 추천하는 것인지?

Purpose

  • 추천의 목적.
  • 상품을 구매하도록 유도하는 것
  • 사용자를 교육하는 것 : Word Processor의 Tip 기능, TripAdvisor의 호텔 추천
  • 유저들간의 커뮤니티를 형성하는 것 : SNS(Referral Web)

Recommendation Context

  • 추천하는 상황, 맥락 : 유저가 추천을 받을때 무엇을 하고 있는지?
  • 음악을 듣고있는지, 쇼핑중인지, 다른 사람과 교류중인지 등을 고려해야 함

Whose Opinions

  • 추천이란 것은 누군가의 의견을 제시해주는 것임
  • 전문가? 모든 사람? 유저와 비슷한 사람?

Personalization Level

  • Generic / Non-Personalized : 모든 사람이 같은 추천을 받음
  • Demographic : 특정 그룹(성별, 나이 등)에 맞춰서 추천
  • Ephemeral : 현재 활동에 의한 추천
  • Persistent : 관심 및 선호도에 의한 추천

Privacy and Trustworthiness

  • Privacy : 추천을 할 시, 개인정보를 너무 자세히 알고 있다고 생각할 수 있다.
  • Trustworthiness: 추천이 얼마나 정직한지? 추천자가 자기 이득을 위해 정직한 추천을 하지 않을 수 있음

Interfaces

  • Input(추천시 사용하는 정보)

    • Explicit
    • Implicit

  • Output(추천의 종류)

    • Predictions
    • Recommendations
    • Filtering
    • Organic vs. Explicit presentation

Recommendation Algorithms

Basic Model
  • Users : 추천 받을 사용자
  • Items : 추천할 아이템
  • Ratings : 사용자가 아이템을 좋아하는지? 선호도.
  • (Community) : 사용자들의 의견/선호가 모이는 공간
Non-Personalizer Summary Statistics
  • Community 외부 정보 사용 : 베스트 셀러, Trending-Hot
  • Summary of Community Ratings : Best-liked
  • 빌보드 랭킹, TripAdvisor 호텔 랭킹 등이 있
Content-Based Filtering
  • User Ratings x Item Attributes => Model
  • 사용자가 부여하는 평점, item의 속성을 이용해 User Model을 만듬
  • ex) 사용자가 Action 영화(Item attribute)에 계속 좋은 평점(User Rating)을 줬다면 User Model 이 사용자가 어떤 배우를 좋아하는지 등을 추천 가능
  • 개인 뉴스피드, 아티스트별/장르별 뮤직 피드
Personalized Collaborative Filtering
  • 다른 사람들의 의견을 바탕으로 예측/추천
  • User model, Item model : set of ratings
  • (U, I) 행렬을 rating으로 채우고 빈 부분을 채움
Collaborative Filtering
  • User - user : 비슷한 사람들을 묶어서 그 사람들의 의견을 바탕으로 추천
  • Item - item : 비슷한 item끼리 묶어서 각 item의 평점에 따라 비슷한 item들도 추천
  • Dimensionality reduction
Others
  • Interactive recommenders : critique-based, dialog-based
  • Hybrids of various techniques
